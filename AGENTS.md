# AGENTS.md

## Назначение

Этот файл задаёт правила для контрибьюторов и AI-агентов в репозитории `Capitask`.

## Open-source

- Проект открыт для сообщества и развивается как open-source.
- Любые изменения должны быть понятны внешнему контрибьютору.
- В PR обязательно описывать цель, влияние на поведение и риски.

## Базовые стандарты разработки

- Писать простой, читаемый код.
- Избегать «магических» значений, выносить их в константы.
- Для нетривиальной логики добавлять короткие комментарии на русском языке.
- Не добавлять лишние зависимости без обоснования.

## Безопасность (обязательно)

- Считать безопасность частью definition of done для любой задачи.
- Валидировать и санитизировать входящие данные на клиенте и сервере.
- Не хранить секреты в репозитории.
- Не логировать токены, пароли, персональные и другие чувствительные данные.
- Для auth и access-control изменений требуются:
  - отдельное описание угроз;
  - проверка граничных кейсов;
  - ревью минимум одним разработчиком.

## Документация

- При изменении поведения обновлять `README.md`.
- При изменении требований к backend обновлять `docs/backend-requirements.md`.
- Любой новый публичный API должен быть документирован.

## Проверки перед merge

- Приложение запускается локально.
- Сборка проходит без ошибок.
- Нет явных регрессий по основным пользовательским сценариям.
- Для важных изменений добавлены/обновлены тесты.
